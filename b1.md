# Bài 1: Phân tích & Lựa chọn prompt tối ưu

## Đáp án lựa chọn
**Phương án B**

## Vì sao B là tối ưu nhất theo 5 thành phần Prompt

### 1. Vai trò
Phương án B yêu cầu AI **đóng vai trò là một Java Senior Developer**. Đây là điểm quan trọng vì AI sẽ trả lời theo góc nhìn của người có kinh nghiệm, biết ưu tiên tính đúng nghiệp vụ, tính dễ đọc và chuẩn Clean Code.

### 2. Mục tiêu
Phương án B nêu rõ mục tiêu là **refactor logic rẽ nhánh lồng nhau thành guard clauses / return sớm**. Mục tiêu càng cụ thể thì AI càng ít suy đoán và càng dễ tạo ra đúng kiểu code mong muốn.

### 3. Ngữ cảnh
Prompt đã cung cấp đúng bài toán: class `DiscountService`, logic tính chiết khấu, và yêu cầu **giữ nguyên nghiệp vụ cũ**. Có ngữ cảnh đầy đủ nên AI hiểu cần sửa phần nào và không tự ý đổi bài toán.

### 4. Ràng buộc
Phương án B có các ràng buộc rất rõ:
- Giữ nguyên logic nghiệp vụ ban đầu
- Không đổi kiểu dữ liệu đầu vào/đầu ra
- Dùng Java 11
- Trả kết quả dạng code Java hoàn chỉnh
- Có giải thích ngắn bằng tiếng Việt

Nhờ vậy, AI có “ranh giới” rõ ràng để không làm sai yêu cầu.

### 5. Định dạng
Prompt B yêu cầu đầu ra là **khối mã nguồn Java hoàn chỉnh kèm giải thích ngắn**. Điều này giúp kết quả dễ nộp bài, dễ đọc, và đúng với yêu cầu của đề.

## Vì sao loại các phương án còn lại

### Phương án A
"Tái cấu trúc code Java trên để nó đẹp hơn." 

Nhược điểm:
- Quá chung chung, không nói rõ phải dùng guard clauses
- Không nêu giữ nguyên nghiệp vụ
- Không có ràng buộc về Java 11 hay đầu ra
- AI có thể chỉ format lại code chứ không refactor logic

### Phương án C
"... sử dụng cấu trúc Java Stream API ..."

Nhược điểm:
- Sai định hướng bài toán vì đề yêu cầu **guard clauses**, không phải Stream API
- Có nguy cơ làm code phức tạp hơn với bài toán điều kiện rẽ nhánh
- Dễ làm lệch nghiệp vụ nếu cố ép dùng Stream không phù hợp

## Kết luận ngắn
**Chọn B** vì prompt này đủ 5 thành phần cần thiết: có vai trò rõ, mục tiêu rõ, ngữ cảnh rõ, ràng buộc rõ và định dạng đầu ra rõ. Đây là prompt an toàn nhất để AI refactor đúng yêu cầu mà không làm sai logic chiết khấu.
